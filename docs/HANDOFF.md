## 游戏方向

项目方向已从抽象经营按钮游戏，调整为场景化网吧经营模拟。

核心体验：

```text
客人从入口进门
-> 到前台开机
-> 走到电脑上机
-> 上机读条
-> 随机点商品/去厕所
-> 下机离场
-> 产生收入和清洁压力
```

目标风格：

```text
温暖像素风，参考星露谷物语的室内经营氛围。
```

## 已实现功能

- 开局小黑网吧场景：入口、前台、4 台大厅电脑、厕所门。
- 客人自动进店、排队、前台开机、上机、离场。
- 上机时间已放慢，便于观察场景运转。
- 顾客上机期间会随机提出商品需求。
- 采购系统：泡面、矿泉水、烤肠、槟榔、香烟等商品。
- 仓库系统：查看当前库存，可从仓库进入采购。
- 商品解锁受网吧等级限制。
- 初始测试资金改为 10000，每次刷新都会从 10000 开始。
- 清洁值系统：顶部温度计显示清洁度。
- 客人下机后电脑变脏，出现清洁提示。
- 可手动点击脏电脑清洁。
- 厕所门会随机被客人使用，厕所会变脏，可手动清洁。
- 招聘系统：收银员、外场、保洁、店长、陪玩。
- 员工招聘后会出现在场景里。
- 员工统一穿黑黄围裙，用不同颜色帽子区分岗位。
- 收银员站前台，可加快开机。
- 外场/保洁/店长会自动清理脏电脑和厕所。
- 收银员/店长会在有库存时自动给顾客配送商品。
- 店长会自动补货，没店长时只能手动采购。
- 设备系统：当前开局为 1080+i3。
- 设备档位：
  - 1080+i3
  - 2080+i5，3000/台
  - 3080+i7，5000/台
  - 4080+i9，7000/台
  - 5080+R9，10000/台
- 设备升级已改为指定机器升级。
- 每台电脑有独立设备档位，场景上显示类似 `1号 L1`。
- 设备面板点升级后会弹出机器选择列表。
- 网吧等级暂定最高 Lv.4，按员工、设备、机器数量综合计算，不走经验值。

## 当前已知问题/注意事项

- 中文在 PowerShell 里有时显示乱码，但微信开发者工具里通常能正常显示；新增中文文案建议尽量用 Unicode 转义，减少编码问题。
- 目前没有持久存档，刷新会重置所有状态。这是刻意保留的测试状态。
- `project.config.json` 被微信开发者工具自动修改，暂时不要提交。
- 目前员工没有工资结算周期，只显示月薪数值。
- 店长自动补货会直接扣现金，后续需要更明确的补货策略和提示。
- 网吧等级规则目前是临时规则，需要后续和机器数量、包厢、设备、员工系统一起重构。
- 设备升级现在按单台升级，但包厢/区域系统还没做，后续应给机器增加区域字段。

## 下一步建议

优先级建议：

1. 做存档开关或测试模式开关。现在每次刷新 10000 适合测试，但后续需要区分正式玩法。
2. 做机器区域字段，为后续大厅、多人包厢、单双包厢、胶囊房铺路。
3. 做员工工资结算和班次系统。
4. 做顾客类型：低端散客、高端玩家、包厢客、陪玩客。
5. 做上机价格设置：大厅 5 元/小时，可调价影响客流。
6. 做设备对顾客类型和消费概率的影响。
7. 做店长自动补货策略 UI，避免玩家感觉现金无故减少。
8. 优化美术比例和场景动线，继续向温暖像素经营风靠近。
